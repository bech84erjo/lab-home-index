# lab-home-index

本仓库用于归档和发布多个独立的 HTML 页面。这些页面不针对任何特定域名或具体网站，而是作为实验性、演示性或辅助性内容的静态页面集合。

---

## 项目简介

`lab-home-index` 是一个轻量级的 HTML 页面归档仓库。其目的是将多个零散的、独立的 HTML 文件集中存放，便于版本管理、浏览和分享。仓库本身不提供动态服务，仅作为静态内容的存储和发布源。

适合存放：
- 单页工具或小应用
- 演示或原型页面
- 学习笔记或文档页
- 临时测试页

---

## 目录结构

```
lab-home-index/
├── index.html          # 仓库首页（可选的页面索引）
├── page1/              # 示例页面目录
│   └── index.html
├── page2/
│   ├── index.html
│   └── assets/
├── tools/              # 工具类页面
│   └── ...
├── demos/              # 演示类页面
│   └── ...
└── README.md           # 本文件
```

- 每个独立页面建议放置在自己的子目录中，以便包含相关资源（CSS、JS、图片等）。
- 根目录下的 `index.html` 可作为导航或入口页，列出仓库内所有页面链接（可选）。
- 目录命名应简洁、有意义，便于理解页面内容。

---

## 页面归档说明

- **格式要求**：所有页面应为标准的 HTML 文件，编码统一使用 UTF-8。
- **资源引用**：建议使用相对路径引用资源，确保页面在本地或 GitHub Pages 上都能正常显示。
- **兼容性**：页面应尽量兼容主流浏览器，无需依赖特定环境。
- **独立性**：每个页面应尽可能独立，不依赖其他页面或外部服务（除非必要）。

---

## 维护说明

### 添加新页面

1. 在仓库根目录下创建新文件夹（如 `new-page/`）。
2. 将页面文件及相关资源放入该文件夹。
3. 如有需要，更新根目录下的 `index.html` 添加链接。
4. 提交 Pull Request 或直接推送（视仓库权限而定）。

### 更新页面

- 直接修改对应目录下的文件即可。
- 请保留历史版本，避免破坏已有链接。

### 删除页面

- 删除对应目录后，记得更新 `index.html` 移除相关链接。
- 删除操作不可逆，请确认页面不再需要。

## 使用方式

您可以通过以下方式访问仓库中的页面：

- **本地浏览**：克隆仓库后，直接在浏览器中打开 HTML 文件。
- **GitHub Pages**：如果仓库启用了 GitHub Pages，可以通过对应的 Pages URL 访问页面（具体 URL 请参考仓库设置）。
